Proyecto "ExploraChia"

Ingenieria de datos

1. Selección de Datos

Google Maps se identificó como una fuente valiosa de datos debido a los comentarios públicos de los usuarios sobre sitios de entretenimiento. Utilizando Apify, una plataforma de web scraping, se extrajeron datos clave como la fecha, hora y calificación de los comentarios a un formato CSV.

  • publishedAtDate: Fecha y hora en la que se publicó el comentario (tomada como día de visita)
  • publishAt: "Texto relativo (ej. ""Hace 2 horas"") – solo informativo"
  • stars: Calificación del sitio (1 a 5 estrellas)

2. Limpieza de Datos

Gestión de valores faltantes

Se verificó que no hubiera valores faltantes en las columnas clave, como la fecha de publicación o las calificaciones de los comentarios. Si alguno estaba ausente, se aplicaron técnicas de imputación o eliminación, según fuera necesario.

Filtrado de datos irrelevantes

Se filtraron aquellos registros que no aportaban valor al análisis o eran obvias (por ejemplo, comentarios o ubicación) para asegurar que solo se utilizara información útil.

3. Transformación de Datos

Creación de nuevas características

A partir de los datos extraídos, se generaron nuevas columnas útiles:

  • Día de la semana: Se extrajo el día de la semana (Lunes, Martes, etc.) de la fecha de publicación para ayudar a predecir el comportamiento de las visitas según los días.
  • Hora: La hora de publicación se extrajo para analizar si la hora influye en la cantidad de visitas.
  • Fecha: Se aisló la fecha de la publicación para agrupar los datos por día.

Normalización

Se transformaron los datos de las categorías (por ejemplo, día de la semana) a valores numéricos para que el modelo de Machine Learning pudiera procesarlos fácilmente.

4. Documentación

Todas las transformaciones de datos fueron documentadas en un archivo Word, incluyendo líneas de código empleadas para conversión, agrupación, y creación de nuevas variables, garantizando trazabilidad y reproducibilidad.

E igualmente de forma colaborativa con la creación de ramas y commits en un repositorio en GitHub. La documentación estructurada nos permite replicar el proceso, facilitar auditorías y mejorar el trabajo colaborativo.

Ingenieria del modelo

Metricas de Desempeño

Para evaluar el rendimiento de los modelos de clasificación utilizados en este proyecto, se seleccionaron métricas clave que permiten analizar su precisión y capacidad predictiva. Entre estas se incluyen la matriz de confusión, que muestra la relación entre predicciones correctas e incorrectas; la precisión, que indica la proporción de verdaderos positivos entre las predicciones positivas; la sensibilidad o recall, que mide la capacidad del modelo para identificar correctamente los casos positivos; y el F1-score, que equilibra precisión y sensibilidad en una sola métrica. Estas métricas ofrecen una visión integral del comportamiento del modelo frente a los datos real

Validacion Cruzada

En este proyecto se utilizó validación cruzada de 5 particiones para estimar el error del modelo en nuevos datos, asegurando que las métricas reportadas no dependan de una única división entrenamiento/prueba.

3. Análisis de Riesgos

Se identificaron posibles riesgos relacionados con el sobreajuste, en el que el modelo aprende demasiado bien los datos de entrenamiento y pierde capacidad de generalización. También se consideró el subajuste, que ocurre cuando el modelo no logra captar patrones importantes en los datos, lo que afecta su rendimiento.

3.1. Subajuste

Modelo con muy pocas variables predictoras

Al inicio del proyecto, los modelos solo utilizaban la fecha o el número de comentarios diarios como entrada. Esto no capturaba información importante como el día de la semana, el mes o si era temporada alta, lo cual limita la capacidad del modelo para aprender patrones relevantes

Modelo muy simple (por ejemplo, Regresión Lineal básica)

En etapas tempranas, si se usó un modelo demasiado simple (como una regresión sin variables categóricas o sin interacciones), no se logró captar relaciones no lineales en los datos.

Entrenamiento con pocos datos

Algunos sitios tienen menos de 100 comentarios, lo cual puede haber llevado a un modelo poco entrenado que no generaliza bien.

3.2. Sobreajuste

Modelo ajustado a series con pocos datos y ruido

Si se entrenó un modelo muy complejo (como Random Forest con muchos árboles) sobre un conjunto de datos pequeño o muy específico de un sitio, este pudo aprender los “picos” de visitas de manera exacta, pero fallar al generalizar

Uso excesivo de variables irrelevantes o correlacionadas

Si se incluyeron muchas variables (por ejemplo, múltiples dummies por mes, sin regularización o selección), el modelo podría haber aprendido patrones espurios.

Entrenamiento sin validación cruzada adecuada

Si no se separaron bien los datos de entrenamiento y prueba, o no se usó una validación temporal, el modelo pudo haber aprendido los datos de entrada sin poder generalizar a nuevas fechas.

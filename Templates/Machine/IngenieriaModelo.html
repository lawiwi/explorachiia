Ingenieria del modelo

Preparación de los datos

Se dividieron los datos en características (X) y objetivo (y). Las características incluyen variables como el día de la semana (representado numéricamente) y la hora, mientras que el objetivo fue el número de visitas (visitas por día).

División de Datos

Se dividió el conjunto de datos en dos subconjuntos: 80% para entrenamiento y 20% para prueba. Esto permitió evaluar el desempeño del modelo en datos no vistos durante el entrenamiento.

Entrenamiento del Modelo

Utilizando la técnica Random Forest Regressor, se entrenaron 100 árboles de decisión. Este proceso consistió en generar múltiples modelos (árboles) con subconjuntos aleatorios de los datos de entrenamiento, lo que permitió una mayor precisión y robustez.

Ajuste de Parámetros

El modelo se ajustó con los parámetros predeterminados de Random Forest, como el número de árboles (n_estimators=100) y la semilla aleatoria para asegurar resultados consistentes (random_state=42).

Selección de Algoritmos

Se eligió Random Forest debido a su capacidad para manejar tareas complejas y su excelente rendimiento en modelos predictivos. Es adecuado por su capacidad de adaptarse a diversos tipos de datos y su robustez frente al sobreajuste.

Escalabilidad

Capaz de manejar grandes cantidades de datos y múltiples características sin pérdida significativa de rendimiento, ideal para datos de diferentes sitios y horarios.

Precisión

Captura relaciones no lineales entre características y el objetivo, ideal para predecir visitas basadas en patrones temporales y otras variables.

Robustez

Al trabajar con múltiples árboles de decisión, es menos susceptible al sobreajuste y generaliza mejor incluso con datos ruidosos o desequilibrados.
